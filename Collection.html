INTRODUCTION

  • A collection is a container that stores and organizes other objects.
  • The object stored inside are called items and element.
  • Most important concept in data structure mostly focused on Grouping and Managing Data.
  • Collections are sometimes also called data structures or abstract data types when we describe their behavior independently of how they are implemented.
  • The examples of the collections [list, array, stacks, queue and etc] you will see them through out the process.

Type of Collection

  1. Linear Collection

    • Elements are arranged in sequence form where every item has a predecessor and a successor, except the first and last items. The last item has only a predecessor, and the first item has only a successor. Examples include lists, stacks, and queues.

    • For Example:

    • [A, B, C, D]{Linear Collection}

      • A → first item → has successor B only.
      • B → has predecessor A and successor C.
      • C → has predecessor B and successor D.
      • D → last item → has predecessor C only.
  2. Hierarchical Collection
    • Where item are arranged in a parent like relationship (tress).
    • In a tree each item except the root has exactly one parent, and items can have zero or more children,forming levels from top (root) to leaves.

  3. Graph Collection
    • Items can be connected to many other items — each item can have many predecessors and many successors at the same time..
    • An item (vertex) can have edges to many other vertices.
    • There may be cycles, so there is no single root or strict parent–child hierarchy.
    • All elements connected to a given vertex are considered both its predecessors and its successors — these are called its neighbors.

  4. Unordered Collection
    • items has no position on a sequence(bags, sets).
  5. Sorted Collection
    • Items are arranged in ascending order or descending order.
    • Does not have to be linear or ordered by position.
    • Sets, bags, and dictionaries can also be sorted — even though their items are not accessed by position/index.
    • A binary search tree (a hierarchical collection) also imposes a natural ordering on its items.

OPERATION OF COLLECTION

  • Every collection regardless of the type they support set of operation.
  • Operation allows to interact with data inside.

Operation

  • isEmpty() → check if the collection is empty.
  • len() → return the number of item in a collection.
  • str() → return string representation.
  • in → check if an item exist in the collections.
  • + → joins two collections together into a new collection containing all their elements in order.
  • = → operator checks whether two collections contain the same elements in the same order. If they do, the collections are equal.
  • iter() → allow you to loop through the collections.

    • These operations form part of the abstract interface of a collection; different implementations
    must still support these behaviors even if the internal code is different    .

Type of Conversion

  • Converts one type of collection into another type.
  • Not limited to collections only — any iterable object can be converted into a collection.
    • Example: list(range(1, 11, 2)) → [1, 3, 5, 7, 9] — a range is iterable but not itself a collection.
  • Common examples:
    • list(collection) → converts to a list
    • tuple(collection) → converts to a tuple
    • dict(collection) → converts (key, value) pairs into a dictionary
    • sorted(collection) → returns a sorted list of a collection's items
  • If the argument is omitted, the conversion function returns a new, empty collection of that type.

    • Cloning and Equality

    • Cloning is a special case of type conversion — the source and new collection are the same type.
    • Creates a copy of a collection (changing the clone does not change the original).
    • Cloning produces a shallow copy
      • Items themselves are not cloned — only references to them are copied.
      • Fine for immutable items (numbers, strings, tuples).
      • Risky for mutable items shared between original and clone — changing one can affect the other.
      • To avoid this, use a deep copy — loop over the source collection and explicitly clone each item before adding it.
    • Equality (==) checks if two collections have the same items in the same position.
      • Two collections are structurally equal when they have the same number of elements in the same order — even if stored in different memory locations.
    • Identity (is) checks whether two variables refer to the exact same object in memory — different from ==.
      • lyst1 = [2, 4, 8]
      lyst2 = list(lyst1) # clone of lyst1
      lyst1 is lyst2 # False — two distinct objects
      lyst1 == lyst2 # True — same type, same items, same order
