Chapter 2 - Algorithm and Complexity Analysis

INTRODUCTION

What is an Algorithm

A step-by-step procedure or set of rules designed to solve a problem or perform a task.

Every algorithm have important properties:

  1. Correctness – it must produce the right result for all valid input.
  2. Efficiency – it must use a reasonable amount of Time and Memory.
  3. Clarity – it must be understandable to other programmers.

When we talk about the performance on an algorithm we concerned with 2 resources:

  1. Time – How long it take to run.
  2. Space – how much memory it uses.

Time alone is not a reliable measure:

  • You can have the same code but different machine and time depend on how good is your machine, so if your machine is good the time complexity will be low.

Instead, use complexity analysis:

  • This Method is independent to Machine, time and any other factor

Complexity Analysis

  • Measure growth of work
  • Independent to Machine(something at you are using to run you code like you pc etc)
  • It uses Big – O notation

Big – O notation

  • How you express a complexity of an algorithm
  • Describe how the work of an algorithm does grow as input size n increase
  1. O(1) Constant
    • work
  2. O(logn) Logarithm
    • Work grows Slow
    • e.g Binary Search
  3. O(n) Linear
    • Work grow with n
    • e.g sequential Search
  4. O(n logn)
    • slight above the line
    • e.g merge sort
  5. O(n^2) quadratic
    • Work grow as n squared
    • e.g bubble sort, Selection Sort
  6. O(2^n) exponential
    • Work explode
    • recursive Fibonacci

  • Best to Worst performance:
    • O(1) → O(logn) → O(n) → O(n logn) → O(n^2) → O(2^n)
  • Cases for Complexity analysis:
    • Best case – the scenario where the algorithm does the least work to finish.
    • Worst case – the scenario where the algorithm does the most work to
      finish — this is the one most commonly analyzed, since it gives a guaranteed upper bound.
    • Average case – the expected amount of work across all possible inputs, not just one extreme.

⚠️ Important nuance not to skip: these three cases only produce different complexities when the algorithm's work actually depends on the arrangement of the input (e.g. it can exit early, or skip ahead). If an algorithm always does the same fixed amount of work regardless of input, best/worst/average case are identical

1. Searching for the Minimum

📝The Idea:

  • We want a function that doesn’t just give us the smallest value in a list, but instead tells us where (the index) that smallest value is.

🔎 How It Works

  1. Start at the beginning Assume the first item (index 0) is the smallest.
  2. Scan through the list Move from left to right, checking each item one by one.
  3. Compare If you find an item smaller than the current "smallest," update your record:
    • Save the index of this new smaller item.
  4. Finish When you reach the end, return the index of the smallest item found.

⚡ Example

python

def indexOfMin(lst):
# Step 1: Assume first item is smallest
min_index = 0
# Step 2: Check the rest of the list
for i in range(1, len(lst)):
# Step 3: Compare
if lst[i] < lst[min_index]:
min_index = i
# Step 4: Return the index
return min_index

📊Walkthrough Example

Suppose to have:

python
numbers = [8, 3, 5, 2, 7]

  • Start: min_index = 0 (value = 8)
  • Compare index 1 → 3 < 8 → update min_index = 1
  • Compare index 2 → 5 < 3? No → keep min_index = 1
  • Compare index 3 → 2 < 3 → update min_index = 3
  • Compare index 4 → 7 < 2? No → keep min_index = 3

✅Final answer: 3 (the smallest value is at index 3, which is 2).

Complexity

  • The algorithm must check every item once to be sure it has found the true minimum.
  • Comparisons: always n − 1, regardless of the data.
  • Time complexity = O(n)
  • Space complexity = O(1) — you're only ever storing one number (min_index), no matter how big the list gets.

2. Sequential Search

📝The Idea:

  • Sequential search (also called linear search) is the simplest way to look for an item in a list. You start at the first element and check each one in order until you either find the target or reach the end.

🔎 How It Works

  1. Begin at the first item in the list.
  2. Compare it with the target value.
  3. If it matches → return the index (position).
  4. If not → move to the next item.
  5. Continue until the target is found or the list ends.
  6. If the target not found → return something like -1 (meaning "not found").

⚡ Example

python

def sequentialSearch(lst, target):
for i in range(len(lst)):
# Step 1: Compare each item with target
if lst[i] == target:
return i # Found at index i
# Step 2: If loop finishes, target wasn't found
return -1

📊Walkthrough Example

Suppose to have:

python
numbers = [5, 8, 9, 10], target = 9

  • Compare index 0 → 5 == 9? ❌ No
  • Compare index 1 → 8 == 9? ❌ No
  • Compare index 2 → 9 == 9? ✅ Yes → return 2

✅Final answer: 2 (the target is at index 2).

Complexity

  • Unlike indexOfMin, this algorithm can exit early — the moment it finds a match, it stops.
  • Best Case = O(1)
    • target is first.
    • only 1 comparison.
  • Worst Case = O(n)
    • Target is last or missing.
    • All n items checked.
  • Average Case = O(n)
    • n/2 comparisons; constant factor doesn't change growth rate.
  • Space complexity = O(1) — only a few variables used, no extra data structures.

3. Binary Search

📝The Idea:

  • Binary search is a fast way to find an item in a sorted list. Instead of checking one by one, it repeatedly cuts the list in half until the target is found (or the search space is empty).

🔎 How It Works

  1. Start with the entire sorted list.
  2. Look at the middle element.
  3. If the middle element is the target → return its index.
  4. If the target is smaller → search the left half.
  5. If the target is larger → search the right half.
  6. Repeat until the target is found or the list is empty.

⚡ Example

python

def binarySearch(lst, target):
low = 0
high = len(lst) - 1
while low <= high:
mid = (low + high) // 2
if lst[mid] == target:
return mid
elif lst[mid] < target:
low = mid + 1
else:
high = mid - 1
return -1 # Not found

📊Walkthrough Example

Suppose to have:

python
numbers = [2, 4, 6, 8, 9, 11, 13], target = 9

  • Step 1: low=0, high=6 → mid=3 → value=8 → 9 > 8 → search right half.
  • Step 2: low=4, high=6 → mid=5 → value=11 → 9 < 11 → search left half.
  • Step 3: low=4, high=4 → mid=4 → value=9 → ✅ found at index 4.

✅Final answer: 4 (the target is at index 4).

⏱ Time Complexity

  • Unlike sequential search, binary search cuts the problem in half every step — that's what pulls the complexity down from O(n) to O(log n).
  • Best Case = O(1)
    • Target is the middle element on the first check.
    • Only 1 comparison needed.
  • Worst Case = O(log n)
    • Target is at the far end or not in the list.
    • The algorithm keeps halving until the search space is empty.
    • Comparisons: about log2n.
  • Average Case = O(log n)
    • Target is somewhere in the list.
    • On average, it takes about half the number of steps as the worst case.
    • Still expressed as O(log n).

💾 Space Complexity

  • Iterative version: Only a few variables (low, high, mid).
    • Space complexity = O(1).
  • Recursive version:each recursive call adds a frame to the call stack.
    • Space complexity = O(log n).

1. Selection Sort

📝The Idea:

  • Selection Sort organizes a list by repeatedly finding the smallest element and moving it to its correct position. Think of it like arranging cards: you look for the smallest card, place it first, then look for the next smallest, and so on.

🔎 How It Works

  1. Start at the first position.
  2. Search the rest of the list to find the minimum element.
  3. Swap that minimum element with the element at the current position.
  4. Move to the next position and repeat until the list is sorted.

⚡ Example

python

def selectionSort(lst):
n = len(lst)
for i in range(n - 1):
min_index = i
for j in range(i + 1, n):
if lst[j] < lst[min_index]:
min_index = j
# Swap the found minimum with the first unsorted element
lst[i], lst[min_index] = lst[min_index], lst[i]
return lst

📊Walkthrough Example

Suppose to have:

python
numbers = [8, 3, 5, 2, 7]

  • Step 1: Find smallest (2) → swap with first → [2, 3, 5, 8, 7]
  • Step 2: Find smallest in rest (3) → already in place → [2, 3, 5, 8, 7]
  • Step 3: Find smallest in rest (5) → already in place → [2, 3, 5, 8, 7]
  • Step 4: Find smallest in rest (7) → swap with 8 → [2, 3, 5, 7, 8]

✅Final sorted list: [2, 3, 5, 7, 8].

⏱ Time Complexity

  • The algorithm always scans the remaining unsorted portion to find the minimum, on every pass — no early exit exists here.
  • Best Case = O(n²) — even if the list is already sorted, it still searches for the minimum in each pass.
  • Worst Case = O(n²) — reverse order, same number of comparisons and swaps.
  • Average Case = O(n²) — same number of comparisons regardless of input order.
  • 👉 Comparisons: n(n−1)/2. Selection Sort always does the same amount of comparison work, so it's O(n²) in all cases.

💾 Space Complexity

  • Only a few variables are stored (min_index, loop counters).
  • Sorting is done in place — no extra arrays needed.
  • Space complexity = O(1) (constant space).

Summary

  • Best case: O(n²)
  • Worst case: O(n²)
  • Average case: O(n²)
  • Space complexity: O(1)

2. Bubble Sort

📝The Idea:

  • Bubble Sort repeatedly compares adjacent elements and swaps them if they are in the wrong order. The largest values "bubble up" to the end of the list.

🔎 How It Works

  1. Start at the beginning of the list.
  2. Compare each pair of adjacent elements.
  3. Swap them if they are out of order.
  4. Repeat passes until no swaps are needed (list is sorted).

⚡ Example

⚠️ Note: a plain bubble sort with no early-exit check always runs the full nested loop, even on an already-sorted list — it has no way to "notice" the list is sorted and stop. To actually get a linear best case, you need a version that tracks whether any swap happened during a pass.

python

def bubbleSort(lst):
n = len(lst)
while n > 1:
swapped = False
for i in range(1, n):
if lst[i] < lst[i - 1]:
lst[i], lst[i - 1] = lst[i - 1], lst[i]
swapped = True
if not swapped:
return lst # Stop early — list is already sorted
n -= 1
return lst

📊Walkthrough Example

Suppose to have:

python
numbers = [5, 3, 8, 2]

  • Pass 1: [3, 5, 2, 8]
  • Pass 2: [3, 2, 5, 8]
  • Pass 3: [2, 3, 5, 8] → no swaps → stop early

✅Final sorted list: [2, 3, 5, 8].

⏱ Time Complexity

  • Best Case = O(n) — list is already sorted. On the first pass, no swaps occur, the flag stays False, and the function returns immediately.
  • Worst Case = O(n²) — list is in reverse order, maximum number of swaps needed.
  • Average Case = O(n²) — still quadratic on average.
  • 👉 Without the swap-tracking flag, this best case does not exist — a plain bubble sort with no early exit is O(n²) in every case, including best case.

💾 Space Complexity

  • In-place sorting.
  • Space complexity = O(1).

Summary

  • Best case: O(n)
  • Worst case: O(n²)
  • Average case: O(n²)
  • Space complexity: O(1)

3. Insertion Sort

📝The Idea:

  • Insertion Sort builds the sorted list one item at a time by inserting each new element into its correct position among the already sorted elements.

🔎 How It Works

  1. Start with the second element.
  2. Compare it with elements before it.
  3. Shift larger elements one position to the right.
  4. Insert the current element into its correct position.
  5. Repeat until the list is sorted.

⚡ Example

python

def insertionSort(lst):
for i in range(1, len(lst)):
key = lst[i]
j = i - 1
while j >= 0 and lst[j] > key:
lst[j + 1] = lst[j]
j -= 1
lst[j + 1] = key
return lst

📊Walkthrough Example

Suppose to have:

python
numbers = [5, 3, 8, 2]

  • Step 1: Insert 3 → [3, 5, 8, 2]
  • Step 2: Insert 8 → [3, 5, 8, 2] (no change)
  • Step 3: Insert 2 → [2, 3, 5, 8]

✅Final sorted list: [2, 3, 5, 8].

⏱ Time Complexity

  • Unlike plain bubble sort, this algorithm's while loop naturally stops early when it hits an element already in the right place — the early exit is built into the algorithm itself, not an add-on.
  • Best Case = O(n) — list is already sorted; the inner while loop never runs.
  • Worst Case = O(n²) — list is in reverse order; maximum shifting required.
  • Average Case = O(n²) — still quadratic on average.

💾 Space Complexity

  • In-place sorting.
  • Space complexity = O(1).

Summary

  • Best case: O(n)
  • Worst case: O(n²)
  • Average case: O(n²)
  • Space complexity: O(1)

4. Merge Sort

📝The Idea:

  • Merge Sort uses divide and conquer: split the list into halves until each sublist has one element, then merge the sublists back together in sorted order.

🔎 How It Works

  1. Divide the list into two halves.
  2. Recursively sort each half.
  3. Merge the two sorted halves into one sorted list.

⚡ Example

python

def mergeSort(lst):
if len(lst) > 1:
mid = len(lst) // 2
left = lst[:mid]
right = lst[mid:]
mergeSort(left)
mergeSort(right)
i = j = k = 0
while i < len(left) and j < len(right):
if left[i] < right[j]:
lst[k] = left[i]
i += 1
else:
lst[k] = right[j]
j += 1
k += 1
while i < len(left):
lst[k] = left[i]
i += 1
k += 1
while j < len(right):
lst[k] = right[j]
j += 1
k += 1
return lst

📊Walkthrough Example

Suppose to have:

python
numbers = [5, 3, 8, 2]

  • Split → [5, 3] and [8, 2]
  • Sort left → [3, 5]
  • Sort right → [2, 8]
  • Merge → [2, 3, 5, 8]

Final sorted list: [2, 3, 5, 8].

⏱ Time Complexity

  • The list is always split in half, regardless of the data's arrangement — that's why there's no case-to-case variation here.
  • Best Case = O(n log n)
  • Worst Case = O(n log n)
  • Average Case = O(n log n)
  • 👉 Same complexity in all cases, since splitting and merging always happen the same way no matter how the data is arranged.

💾 Space Complexity

  • Needs extra arrays for merging — this is not in-place.
  • Space complexity = O(n).

Summary

  • Best case: O(n log n)
  • Worst case: O(n log n)
  • Average case: O(n log n)
  • Space complexity: O(n)

5. Quick Sort

📝The Idea:

  • Quick Sort is a divide and conquer algorithm. It picks a pivot element, then rearranges the list so all elements smaller than the pivot go to the left, and all larger elements go to the right. It then recursively sorts the left and right parts.

🔎 How It Works

  1. Pick a pivot (commonly the first, last, or middle element).
  2. Partition the list: move all smaller elements to the left of the pivot, and all larger elements to the right.
  3. Recursively apply Quick Sort to the left and right sublists.
  4. Combine results into a fully sorted list.

⚡ Example

python

def quickSort(lst):
if len(lst) <= 1:
return lst
pivot = lst[len(lst) // 2]
left = [x for x in lst if x < pivot]
middle = [x for x in lst if x == pivot]
right = [x for x in lst if x > pivot]
return quickSort(left) + middle + quickSort(right)

📊Walkthrough Example

Suppose to have:

python
numbers = [5, 3, 8, 2]

  • Pivot = 8 → Left = [5, 3, 2], Middle = [8], Right = []
  • Sort Left [5, 3, 2]: Pivot = 3 → Left = [2], Middle = [3], Right = [5]
  • Combine → [2, 3, 5]
  • Final merge → [2, 3, 5, 8]

Final sorted list: [2, 3, 5, 8].

⏱ Time Complexity

  • Unlike merge sort, quicksort's performance depends heavily on how good the pivot choice turns out to be — this is why its cases genuinely differ.
  • Best Case = O(n log n) — pivot splits the list evenly each time.
  • Worst Case = O(n²) — pivot is always the smallest or largest element (bad partitioning, e.g. an already-sorted list with a first/last-element pivot strategy).
  • Average Case = O(n log n) — with reasonably good pivot choices, partitions are fairly balanced.

💾 Space Complexity

  • Recursive calls use stack space.
  • Average space complexity = O(log n).
  • Worst case (bad partitions) = O(n).

Summary

  • Best case: O(n log n)
  • Worst case: O(n²)
  • Average case: O(n log n)
  • Space complexity: O(log n) average, O(n) worst case
